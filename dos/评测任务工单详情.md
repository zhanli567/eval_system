# 评测任务模块

## 需求背景

建设评测任务编排和执行能力，将已发布评测集版本、待评测智能体、自动评估器和人工标签组合成一次可追踪的评测任务，并保存样本级执行结果。

## 需求范围

包含任务列表、任务创建、任务详情、任务启动、任务删除、样本执行结果查看、自动评估结果查看和人工标注。

## 业务规则

1. 任务基础信息
   - 任务名称必填，最长 50 个字符。
   - 描述非必填，最长 200 个字符。
   - 创建任务必须选择评测集和评测集版本。
   - 任务只能选择已发布的评测集版本，不能选择草稿版本。
   - 所选评测集版本必须存在数据行。
   - 创建任务时至少绑定一个评估器或一个标签。

2. 应用配置
   - 应用类型支持 `none`、`agent`。
   - 不关联应用时，应用输出状态记为 `skipped`。
   - 关联智能体时，必须选择智能体应用和应用版本。
   - 智能体入参与评测集字段映射时，同一个应用入参不能重复映射。
   - 应用入参类型支持 `string`、`number`、`boolean`。

3. 评估器绑定
   - 评估器来源支持 `preset`、`custom`。
   - 单个任务最多绑定 5 个评估器。
   - 同一任务内评估器不能重复添加。
   - 预置 LLM 评估器必须在任务中选择模型。
   - 自定义评估器必须选择评估器版本。
   - 评估器必填参数必须完成映射。
   - 参数来源支持 `dataset_field`、`app_output`。
   - 未关联应用时，不允许将参数映射到应用输出。

4. 标签绑定
   - 单个任务最多绑定 5 个标签。
   - 同一任务内标签不能重复添加。
   - 绑定的标签必须存在。

5. 任务状态
   - 任务状态包括 `pending`、`running`、`completed`、`failed`。
   - 任务项状态包括 `pending`、`running`、`annotation_pending`、`completed`、`failed`。
   - 已完成任务不允许重新启动。
   - 正在运行的任务再次启动时直接返回当前详情。
   - 仅 `pending`、`completed`、`failed` 任务允许删除。

6. 执行规则
   - 启动任务后，任务在事务提交后异步执行。
   - 每条样本先调用智能体；未关联应用时跳过智能体调用。
   - 智能体失败时，该样本下评估器结果记为 skipped/failed，不中断整个任务循环。
   - LLM 评估器会渲染 Prompt 并调用模型对话接口。
   - LLM 评估结果要求返回 JSON，至少包含 `score` 字段，可包含 `reason`。
   - 评估分数大于等于通过阈值时结果为 `pass`，否则为 `fail`。
   - Code 评估器当前未接入真实代码执行接口，执行时返回失败结果。

7. 人工标注
   - 标注页展示当前样本、评测集字段、评估器结果和标签配置。
   - 分类/布尔标签通过选项保存，选项分组决定 `pass` 或 `fail`。
   - 数字标签必须填写数值，并按标签阈值计算 `pass` 或 `fail`。
   - 文本标签必须填写文本，提交后按 `pass` 保存。
   - 单条样本的评估器和标签均完成后，任务项变为 `completed`。
   - 所有任务项完成后，任务变为 `completed`。

## API 规则

后端真实接口路径不带 `/api`；前端开发环境的 `/api` 是 Vite 代理前缀。

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/tasks` | 分页查询任务，支持状态、关键字、排序 |
| POST | `/tasks` | 创建任务 |
| GET | `/tasks/{taskId}` | 查询任务详情 |
| POST | `/tasks/{taskId}/start` | 启动任务 |
| DELETE | `/tasks/{taskId}` | 删除任务 |
| GET | `/tasks/{taskId}/items/{taskItemId}/annotation` | 查询样本标注详情 |
| PUT | `/tasks/{taskId}/items/{taskItemId}/annotation` | 保存样本人工标注 |

## 原型修改

需要覆盖任务列表页、创建任务页、任务详情页、任务执行状态展示、样本结果表格、评估器/标签维度统计、人工标注页和上一条/下一条切换。截图由产品侧补充。

## 验收标准

1. 可以分页查看任务，并按状态、关键字和排序条件查询。
2. 可以创建任务，且只能选择已发布且有数据的评测集版本。
3. 创建任务时可以配置智能体字段映射、评估器参数映射和人工标签。
4. 评估器和标签数量、重复添加、必填映射校验符合业务规则。
5. 可以启动任务，任务状态和样本状态按执行结果流转。
6. 智能体失败或单个评估器失败时，失败信息落到样本/评估结果维度。
7. LLM 评估器能解析模型返回的 JSON 评分并计算 pass/fail。
8. 可以查看任务详情、样本数据、应用输出、评估结果和标签结果。
9. 可以进入人工标注页保存各类标签结果。
10. 满足删除条件的任务可以删除，运行中任务不能删除。
