# 评估器管理模块

## 需求背景

建设自动评估器配置能力，用于在评测任务中绑定自动评估维度，通过 LLM 或 code 类型评估器对样本和智能体输出进行评分、通过判断和结果记录。

## 需求范围

包含自定义评估器列表、创建、删除、草稿编辑、版本发布、版本查看，以及预置评估器分类、列表和详情查看。

## 业务规则

1. 评估器基础信息
   - 评估器名称必填，最长 50 个字符。
   - 描述非必填，最长 200 个字符。
   - 评估器类型支持 `llm`、`code`。
   - 评估器创建后不允许修改类型。
   - 创建评估器时自动生成草稿版本，草稿版本 `versionNo = 0`。

2. 列表和删除
   - 支持分页查询自定义评估器。
   - 支持按评估器类型筛选。
   - 支持按评估器名称关键字搜索。
   - 删除评估器为软删除，并同步软删除其版本。

3. 版本管理
   - 草稿版本可编辑。
   - 发布版本只读。
   - 发布草稿时复制当前草稿配置，生成新的发布版本，版本号按 V1、V2 递增。
   - 评估器记录最新发布版本。

4. 评分配置
   - 评分最小值默认 1，最大值默认 5，通过阈值默认 3。
   - 最大值必须大于最小值。
   - 通过阈值必须位于评分范围内。

5. LLM 评估器
   - 必须选择模型。
   - Prompt 必填，最长 2000 个字符。
   - Prompt 必须包含至少一个 `${参数名}` 参数。
   - 参数名从 Prompt 中自动提取，重复参数只保留一次。
   - 参数类型支持 `string`、`number`、`boolean`。
   - 参数描述最长 200 个字符。

6. Code 评估器
   - 执行函数必填，最长 10000 个字符。
   - 必须至少配置一个变量。
   - 变量名必填，最长 64 个字符。
   - 同一版本内变量名不可重复。
   - 参数类型支持 `string`、`number`、`boolean`。
   - 当前任务执行中 code 评估器尚未接入真实代码执行接口，会返回失败结果。

7. 预置评估器
   - 支持查询预置评估器分类。
   - 支持按分类和关键字查询预置评估器。
   - 支持查看预置评估器详情。
   - 创建自定义评估器时可从预置评估器带入默认配置。

## API 规则

后端真实接口路径不带 `/api`；前端开发环境的 `/api` 是 Vite 代理前缀。

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/evaluators` | 分页查询自定义评估器 |
| GET | `/evaluators/presets/categories` | 查询预置评估器分类 |
| GET | `/evaluators/presets` | 分页查询预置评估器 |
| GET | `/evaluators/presets/{presetId}` | 查询预置评估器详情 |
| POST | `/evaluators` | 创建自定义评估器 |
| DELETE | `/evaluators/{evaluatorId}` | 删除自定义评估器 |
| GET | `/evaluators/{evaluatorId}/versions` | 查询评估器版本列表 |
| POST | `/evaluators/{evaluatorId}/publish` | 发布草稿版本 |
| GET | `/evaluators/versions/{versionId}` | 查询版本详情 |
| PUT | `/evaluators/versions/{versionId}` | 编辑草稿版本 |

## 原型修改

需要覆盖评估器列表页、预置评估器入口、创建/编辑评估器页、版本切换、LLM 配置、code 配置、参数配置和发布操作。截图由产品侧补充。

## 验收标准

1. 可以分页查看自定义评估器，并按类型和名称搜索。
2. 可以查看预置评估器分类、列表和详情。
3. 可以创建 LLM/code 两类自定义评估器。
4. LLM 评估器能从 Prompt 自动同步参数。
5. Code 评估器能维护执行函数和变量。
6. 评分范围和通过阈值校验符合业务规则。
7. 草稿版本可编辑，发布版本只读。
8. 发布草稿后生成新的发布版本。
9. 删除评估器后列表不再展示。
