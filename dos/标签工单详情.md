# 标签管理模块

## 需求背景

建设人工标签配置能力，用于在评测任务中绑定人工标注维度，并在任务样本维度保存人工标注结果和通过/失败判断。

## 需求范围

包含标签列表、标签详情、创建标签、编辑标签，以及分类、布尔、数字、文本四类标签的配置规则。

## 业务规则

1. 标签基础信息
   - 标签名称必填，最长 50 个字符。
   - 标签名称不可重复。
   - 描述非必填，最长 200 个字符。
   - 标签类型支持 `category`、`boolean`、`number`、`text`。
   - 标签创建后不允许修改类型。

2. 标签列表
   - 支持分页查询。
   - 支持按标签类型筛选。
   - 支持按标签名称关键字搜索。
   - 展示标签名称、类型、描述、创建时间、更新时间。

3. 分类标签
   - 必须至少配置一个 `pass` 选项和一个 `fail` 选项。
   - 选项名称必填，最长 50 个字符。
   - 同一标签内选项名称不可重复。
   - 选项分组仅支持 `pass`、`fail`。
   - 编辑分类标签时会重建选项配置。

4. 布尔标签
   - 系统固定生成两个选项：`True/pass` 和 `False/fail`。
   - 页面不需要人工维护布尔选项。

5. 数字标签
   - 必须配置最小值、最大值、通过阈值。
   - 最小值、最大值、通过阈值必须为正整数。
   - 最大值必须大于最小值。
   - 通过阈值必须位于最小值和最大值之间。

6. 文本标签
   - 文本标签不维护选项、分值范围和通过阈值。
   - 在任务标注中，文本标签提交后按 `pass` 结果保存。

## API 规则

后端真实接口路径不带 `/api`；前端开发环境的 `/api` 是 Vite 代理前缀。

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/tags` | 分页查询标签，支持 `page`、`size`、`tagType`、`keyword` |
| GET | `/tags/{tagId}` | 查询标签详情 |
| POST | `/tags` | 创建标签 |
| PUT | `/tags/{tagId}` | 编辑标签 |

## 验收标准

1. 可以分页查看标签，并按类型和名称搜索。
2. 可以查看标签详情，详情展示基础信息和类型相关配置。
3. 可以创建四类标签，前后端校验符合业务规则。
4. 标签名称重复时保存失败并提示。
5. 编辑标签时不允许修改标签类型。
6. 分类标签必须同时包含 pass 和 fail 选项。
7. 布尔标签自动生成 True/False 选项。
8. 数字标签按范围和阈值校验。
9. 标签可在评测任务创建和人工标注中被引用。
